# 機能仕様書 (Features)

本アプリに実装されている主要な学習機能とその仕様、裏側で動いているロジックをまとめます。

## トップ画面（Dashboard / Home）
ユーザーの現在の学習状況を視覚的にフィードバックするダッシュボードです。

- **ヒーローエリア**:
  - **ストリーク（連続学習日数）**: 最後に学習した日から翌日までに解答があった場合に加算。2日以上空くとリセットされる。
  - **合格予測スコア**: 全体の正答率（`totalCorrect / totalAnswered`）から算出。
- **学習メニュー（横スクロールカルーセル）**:
  - デイリー、弱点克服、間隔反復、模擬試験などへのショートカットを提供。
- **分野別学習（章一覧）**:
  - 章ごとの進捗グラフ（リングチャート等）とともに表示され、タップすると出題数を選んでからその章（または全章ランダム）の問題を出題。

## 各種学習モード（Quiz Modes）

### 出題数の選択（分野別・弱点克服・間隔反復）
- **UI**: 学習開始前にモーダルで **5問 / 10問 / 20問 / 全問** を選ぶ（初期選択は原則 **10問**。プールが10問未満のときは **5問** または **全問** に合わせる）。
- **ロジック**: 問題プールをシャッフルしたうえで、選んだ件数分を先頭から切り出してセッションに載せる。件数がプールより大きい場合は `min(選択数, プール件数)` となる。
- **対象外**: デイリーチャレンジ（固定5問）、模擬試験（40問固定）は従来どおり。

### 1. デイリーチャレンジ（今日の5問）
- **ロジック**: その日の日付（例: `20260301`）をシード値とした疑似乱数生成器を用いて配列をシャッフルし、先頭の5問を抽出する。
- **目的**: 毎日同じ問題が出題されるため、日課としての学習を促す。

### 2. 弱点克服モード
- **ロジック**: `localStorage` から `weakIds` 配列を読み込み、それに該当する問題だけをフィルターし、上記「出題数の選択」で決めた件数だけシャッフル出題。
- **条件**: このモード内で正解すると、その問題IDは `weakIds` から自動的に削除される。

### 3. 間隔反復（Spaced Repetition）
- **アルゴリズム（SM2ベース変法）**:
  - 各問題に対して `interval`（次回出題までの日数）と `ease`（難易度係数, 初期値2.5）を保持。
  - **正解時**: `interval = Math.round(interval * ease)`, `ease = Math.min(ease + 0.1, 3.0)` に更新。
  - **不正解時**: `interval = 1`, `ease = Math.max(ease - 0.2, 1.3)` にダウン。
- **抽出条件**: `nextReview` タイムスタンプが「現在時刻以前」になっている問題のみをピックアップし、上記「出題数の選択」で決めた件数だけ出題（従来の「最大20問に固定」は廃止）。

### 4. 模擬試験（Mock Exam）
- **ルール**: 全章からランダムに40問をピックアップし、制限時間60分（3600秒）で解答する。
- **タイマー処理**: `setInterval` による1秒ごとのカウントダウン。0になると強制的に結果画面へ遷移する。
- **結果記録**: 70%（28問）以上正解で「合格判定」となり、結果が `mockExams` 配列に記録・成績画面へ反映される。

## 成績・学習データ分析（Stats/Analytics）

### 学習ヒートマップ
- **概要**: GitHubの「草」のようなカレンダーUI。
- **ロジック**: `dailyActivity` の日付別解答数を読み取り、直近90日間の四角形を描画。解答数が多いほど色が濃くなる（CSSによる透明度の制御）。

### 実績バッジシステム
- **条件例**: 「初めての1問解答」「ストリーク7日達成」「模擬試験初合格」などのマイルストーンを `score` や `streak` から判定し、達成時にバッジIDを配列に保存しアンロック表示する。
