# 将来の拡張ロードマップ (Future Roadmap)

現在 GitHub の Issue に残っている未実装の機能案や、将来的なアプリの拡張方向性についてまとめます。

## フェーズ 1: コンテンツの拡充と学習体験の向上
API連携やクラウド化を伴わない、アプリ単体で完結する体験の底上げを行います。

- **用語集のフラッシュカード化**: 「用語集タブ」の単語一覧を、裏表をめくるカードUI（暗記カード）として学習できるモードを追加する。
- **実績バッジの拡充**: 既存の達成条件に加えて、章別制覇・連続正解・週次学習など、継続学習を促すバッジを増やす。
- **学習計画の高度化**: 現在の試験日ベースの学習計画を、苦手章・模試成績・学習履歴に応じてより個別最適化する。
- **PWA（オフライン対応）の強化** (Issue #11): 現在の `sw.js` を拡張し、より堅牢なキャッシュ戦略を導入して「完全オフライン動作」を保証する。
- **テスト自動化の運用強化**: `validate` と Playwright E2E を継続的に回せる CI を整備し、回帰を早期検知できるようにする。

## フェーズ 2: AI連携によるパーソナライズ (Gemini API 活用)
学習者の状況に応じた動的なサポートをAIによって実現します。（Issue #16, #17, #18 関連）

- **AI学習コーチ (Issue #16)**: ホーム画面などの一部にチャットUIを設け、「今は何を勉強すべきか？」「この分野のポイントは？」といった相談にAIが解答状況を踏まえて答えるシステム。
- **解説の深掘りチャット (Issue #18)**: 問題を間違えた際、「解説を読んでもわからない」場合に「ここをもう少し分かりやすく教えて」などとAIに追加質問できる機能。
- **AIによる問題の自動生成 (Issue #17)**: 既存のSQuBOK用語集をベースにして、新しい形式の問題（多肢選択、正誤判定など）をAIが動的に生成し、出題バリエーションを無限に増やす。

## フェーズ 3: データ永続化・クラウド移行 (Firebase 連携)
ローカル完結型から、複数デバイスでの学習継続を可能にするアーキテクチャへの移行です。（Issue #14 関連）

- **Firebase Authentication によるログイン**: Googleアカウントやメールアドレスでの認証を導入。
- **Cloud Firestore によるデータ同期**: 現在 `docs/02_data_model.md` で定義されている `jcsqe_data` のJSONオブジェクトをクラウド上で管理・同期し、PCでやった続きを外出先でスマホから行えるようにする。

---
**※注意点**: フェーズ2（AI連携）およびフェーズ3（クラウド化）は、アプリをGitHub Pagesのような静的ホスティングで運用する場合、APIキーの保護（Firebaseの設定やGemini APIキーの隠蔽などバックエンドや環境変数のセキュアな設計）が必要になります。フロントエンドのみのSPAから、少しアーキテクチャを見直すタイミングとなります。
