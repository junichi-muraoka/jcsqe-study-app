# [Bug] クイズ・結果画面がサイドバーにめり込んで表示される

> この内容を GitHub の New Issue にコピーして起票できます。

## 📌 報告時（発見者が記入）

### 現象
デスクトップ表示（768px以上）で、クイズ画面・結果画面を表示すると、メインコンテンツ（問題文・選択肢）が左サイドバーの下に隠れて表示される。問題文の左端や選択肢が読めない。

### 再現手順
1. デスクトップ幅（768px以上）でアプリを開く
2. 「今日の5問」または「模擬試験」をクリックしてクイズを開始
3. クイズ画面を確認する

### 期待値
クイズ・結果画面はフルスクリーンオーバーレイとして表示され、サイドバーの上に重なって全体が表示される。問題文・選択肢が隠れずに読めること。

### 実際の動作
メインコンテンツがサイドバーの下にめり込み、左側のテキストが隠れて読めない。

### 発生環境
- OS: Windows 10
- ブラウザ: Chrome（デスクトップ表示）

### 発生条件（あれば）
768px以上のビューポート幅（サイドバーが左に表示されるレイアウト）でのみ発生。モバイル表示（ボトムナビ）では発生しない。

### 重大度
- [x] Critical / [ ] High / [ ] Medium / [ ] Low

### 影響範囲
クイズモード全般（今日の5問、弱点克服、間隔反復、模擬試験）、結果画面。デスクトップユーザー全員に影響。

### 回避策（あれば）
ウィンドウ幅を767px以下に縮めてモバイルレイアウトにすると回避可能（非推奨）。

### 添付
スクリーンショットあり（本体がサイドバーに隠れている状態）

---

## 🔧 修正後（修正者が記入）

### 原因
z-index のスタッキングコンテキストによる。`nav.app-nav` は `z-index: 99999`、`div.app` は `z-index: 1`。`#quiz` と `#result` は `.app` の子要素のため、`.app` のスタッキングコンテキスト内で `z-index: 200000` を持っても、親の `.app` が nav より下のレイヤーにあるため、全体が nav の下に表示される。

### 対策
`#quiz` と `#result` を `div.app` の外、`body` の直下に移動。nav と同レベルの要素にすることで、`z-index: 200000` が nav より上に効くようにした。（index.html の DOM 構造変更）

### 検証方法
1. デスクトップ幅でアプリを開く
2. クイズを開始し、問題文・選択肢が隠れず表示されることを確認
3. 結果画面も同様に確認

### 横展開
`#quiz` と `#result` のみが該当。他にフルスクリーンオーバーレイはなし。

### 再発防止
E2E テストに「クイズ画面で問題文がビューポート内に完全表示されていること」のアサーションを追加することを検討。
