# アクセシビリティ方針（a11y）

> **関連 Issue**: [#63](https://github.com/junichi-muraoka/jcsqe-study-app/issues/63)  
> セキュリティ・データの扱いは [security.md](./security.md)。

## 1. 方針

- **キーボード操作**と**視認しやすいフォーカス**を、主要な学習フローで使えるようにする。
- **WCAG 2.x の完全適合**を現時点で宣言はしないが、改善は Issue・PR で積み上げる。
- 参考規格: [W3C WCAG 2.2](https://www.w3.org/TR/WCAG22/)（知覚可能・操作可能・理解可能・堅牢）。

## 2. 実装済み（抜粋）

| 領域 | 内容 |
|------|------|
| **フォーカス表示** | `:focus-visible` でアウトライン（`style.css`。キーボード操作時に視認しやすくする意図） |
| **クイズ画面** | 数字キー **1〜4** で選択肢、**Enter / Space** で「次へ」（解説表示後）、**Esc** でホームへ戻る（`app.js`） |
| **出題数モーダル** | **Esc** で閉じる |
| **用語集フラッシュカード** | **← / →** で前後、**Space / Enter** でめくり、`aria-pressed` / `aria-disabled` 等の付与 |
| **ナビゲーション** | タブ切替ボタンに `aria-current="page"`（アクティブタブ） |

## 3. 今後の改善候補（例）

- 各画面の **見出し階層（h1〜）** の整理と、スクリーンリーダー向け **ランドマーク**
- **色だけに依存しない**正誤表示（アイコン・文言の併用は一部済み。ダッシュボード等の再確認）
- **コントラスト比**の監査（ライト／ダーク両テーマ）
- **スキップリンク**（メインコンテンツへ一発で移動）

## 4. 不具合・要望の出し方

操作性に問題がある場合は [Bug] Issue で、**使用環境（OS・ブラウザ）・キーボードかポインタか・期待する操作**を書いてください（`.github/ISSUE_TEMPLATE/bug_report.md`）。
